DSB2018比赛总结

​ 在两个月前参加kaggle和Booz Allen共同承办的DSB2018比赛，这也是我第一次参加这样的比赛，参赛的原因是因为之前正好看了一些目标检测的论文，想要在比赛上看看最新模型的效果。尽管最后的排名较为一般（第一阶段42/3600+，最终98/3600+），但是在比赛的过程中学习到了很多东西，不管是编程水平、对检测分割任务的理解都有了一定的提高，更重要的是心态上磨练，第一次参赛心态是比较差的，只愿意做能够快速实现然后取得效果的工作，功利心很强，导致后期比赛心态不是很好，这直接导致了第二阶段的成绩下滑严重。希望自己可以吸取教训，保持初心，做有意义有挑战的工作！

​ 拖了两个月，之前一直在忙着复现论文FPN，现在来总结以下排名靠前的队伍的解决方案。

第一名队伍 ods.ai topcoders 的方案总结 地址

主要的创新点：

  • 将任务视为实例分割任务，在预测分割掩码的同时，预测两个细胞核接触的边界。
  • 损失函数: 同时使用交叉熵损失和soft dice loss，认为这么做可以解决分割任务中不同类别的pixel的不平衡问题。
  • 使用足够深的encoder-decoder架构，并且该架构在其他数据集(SpaceNet 和 Inria 等等)上也取得了不错的效果。
  • 使用了一些很特别的后处理技巧，包括watershed(用于分割任务的分水岭算法)、和morphological (形态变化)，并在二级模型上使用GBT算法。
  • 数据增强策略。

改进的过程

  1. 一开始采用了最简单的最简单的unet算法进行分割，对mask标注使用watershed方法进行处理，使得不同的nuclei之间有分割带（保证不同的nuclei之间有一定的间隔），这个方法初始时取得了~.500的分数。
  2. 第二步中，尝试预测nuclei的轮廓，给标注的mask加上轮廓信息，轮廓的大小取决于nuclei的尺寸，标注了轮廓的mask加上watershed的后处理方案让分数提高到了~.525左右分数。
  3. 作者发现对于边界清楚的nuclei, 掩码的轮廓可以预测的很清楚，但是对于边界相接壤的nuceli, 就需要做更多的工作去将其分开。于是作者决定只去预测nuclei的边界，这个改进+watershed的后处理方案使得LB分数到达~.550+左右
  4. 现在有了一个通道预测完整mask，另一个通道负责预测border，但是即使是这样有时候的结果仍然不够好，一个更好的方法是改变nuclei mask的标注信息让边界部分的像素值为空。这使得我们可以使用softmax函数作为预测时的激活函数去替换sigmoid。这么做使得细胞核可以更好的分开，但是却降低了mAP, 原因是在IOU阈值较高时的得分不够高。作者解决的方法是另外使用完整mask 标注训练一个网络，然后将后处理时将两者的结合起来。
    （这段话并不是很明白，1）不是已经有一个通道使用完整的mask标注来训练了么？2）为什么让了nuclei的边界像素值为空就能够使用softmax函数作为激活函数了？

数据增强策略

  • Clahe, Sharpen, Emboss
  • Gaussian Noise
  • Color to Gray
  • Inverting - we should not have used it, some images were not predicted correctly on stage2 because of this augmentation
  • Remapping grayscale images to random color images
  • Blur, Median Blur, Motion Blur
  • contrast and brightness
  • random scale, rotates and flips
  • Heavy geometric transformations: Elastic Transform, Perspective Transform, Piecewise Affine transforms, pincushion distortion
  • Random HSV
  • Channel shuffle - I guess this one was very important due to the nature of the data
  • Nucleus copying on images. That created a lot of overlapping nuclei. It seemed to help networks to learn better borders for overlapping nuclei.

网络结构

​ 基本架构使用的UNet模型，作者发现VGG16在这个这个任务上的效果并不是很好，决定让网络朝着更深的方向发展。实验之后作者发现效果比较好的几个网络分别是：DPN-92, Resnet-152， InceptionResnetV2, Resnet101

二级模型/后处理

​ 使用LightGBM模型来判断预测出的nuclei是否为真的nuclei，该模型为一个回归模型，具体预测值为该nuclei和grond truth的IOU值，之后根据预测的IOU值选出一个合适的阈值将阈值低的nuclei排除。而输入的特征主要选用了一些形态学上的特征，包括 solidity, circularity, convexity, area, neighbors median area, count。（真的强！）

第五名队伍 Inom Mirzaev 方案总结 地址

​ 采用 Matterport’s 的Mask R-CNN模型，使用额外的数据集 external dataset 并加上大量的数据增强方案。除此之外，还通过对原始数据集的拼接去产生的新的数据，基本方案为 this notebook。为了防止过拟合，将原始数据集分为训练集和验证集（基于Allen Goodman 对训练图像的所分的五个类别。在使用ResNet101作为网络主架构后，通过TTA和对mask的后处理提高精确度

以下为作者的说明，给值得注意的地方加粗。

Training Method(s)

Pre-processing
  • I noticed some issues with the provided masks. Therefore, used the annotations and mask provided by Konstantin Lopuhin in data quality issues thread.
  • Removed the alpha channel from the images.
  • Filled holes in the masks
  • Splitted (using stratification) the stage1_train dataset into 90% train and 10% validation sets based on 5 image modalities provided by Allen Goodman.
  • Used an external dataset provided in the forum. Divided the images and the masks into 4 pieces due their large sizes. External dataset download links.
  • Generated mosaics from train images based on Emil’s notebook.
Model and Training
  • Modified Matterport’s implementation of Mask-RCNN deep neural network for object instance segmentation.
  • Tuned hyperparameters to detect small nuclei from the images. (I found this tutorial very useful for understanding the model hyperparameters)
    • Original Matterport implementation was validating only on one image so fixed this validation issue.
    • Reduced RPN (region proposal network) anchor sizes since the nuclei are mostly small.
    • Increased number of anchors to be used since the nuclei are small and can be found anywhere on an image.
    • Increased maximum number of predicted objects since an image can contain 300 or more nuclei.
    • Increased POST_NMS_ROIS_TRAINING to get more region proposals during training.
    • Added extra parameter DETECTION_MASK_THRESHOLD to model configuration. Default was hardcoded in the model as 0.5 but setting it to 0.35 helped in detection of small nuclei boundaries.
    • Resized images and masks to 512x512
  • Relied heavily on image augmentation due to small training set:
    • Random horizontal or vertical flips
    • Random 90 or -90 degrees rotation
    • Random rotations in the range of (-15, 15) degrees
    • Random cropping of bigger images and masks to 256x256x3.
    • Random scaling of image and mask scaling in the range (0.5, 2.0)
  • Used Resnet101 architecture as a backbone encoder but initialized the first 50 layers of the model with pre-trained Resnet50 weights from ImageNet competition.
  • Trained the model with Adam optimizer for 75 epochs:
    • 25 epochs with learning rate 1e-4
    • 25 epochs with learning rate 1e-5
    • 25 epochs with learning rate 1e-6
  • Did not retrain the model with stage1 test data during stage 2 as I was afraid of overfitting on black and white images.
Post-processing
  • Combined predictions on actual image and horizontally flipped image: took unions of masks with maximum overlap and removed false positive masks with small overlap.
  • Due to configured RPN anchor sizes, the model predicts small nuclei very well. However, it struggles at predicting large nuclei. Therefore, if a model predicts no masks for an image then I scale down the image and predict once again. （我也这么做了- -）

  • Removed overlaps between predicted nuclei based on their objectness score. In other words, removed intersections from the masks with lower scores.

    • If this intersection removal results in multiple objects in that mask, then removing all the small pieces.（如果多个object之间有mask重合的部分，则将这部分mask从得分不是最高的object上删去

  • Closing small holes inside the masks using morphological operations (dilation followed by erosion).

Interesting findings
  • Mask-RCNN model overfits easily without image augmentation.
  • Removing false positive mask predictions improves the overall score significantly.
  • Since images are on different scales, predicting masks on scaled images helps with the model generalizability.
  • Dilating and then eroding individual masks helped me achieve slightly better result.
  • Matterport’s original implementation was only validating on only one image. Fixing this issue made the training process reproducible.
  • I found that the model reaches a local minima faster when trained using Adam optimizer compared to default SGD optimizer.
Unsuccessful approaches tried
  • Trained the model with Dice Coefficient Loss instead of default binary cross-entropy loss for the masks heads. Although got comparable results couldn’t beat prediction accuracy of binary cross-entropy on my validation set.
  • Trained with random Gaussian and Poisson (or shot) noise for image augmentation. It actually hurt overall model performance.
  • Tried ensembling actual image predictions with horizontal and vertical flip predictions. Used non-maximum suppression for removing overlaps. Did not improve prediction accuracy on the validation set.
  • Trained end-to-end without initializing with pre-trained ImageNet weights. Mostly got to Mean IoU score of 0.35 on stage1 test set.
  • Trained on preprocessed images with adaptive histogram equalization (CLAHE). The model performed way worse.

矩阵微分法和常用符号标记规范

​ 之前的心态一直都很急，只想着做能够快速见到成果的事，很多知识掌握得模棱两可就放过去了，也尝到了很多苦头，需要不停得返回去看。于是痛定思痛，下定决心用博客记录下自己学习的过程，认真的对待自己学过的东西。之前一直感觉全连接网络和卷积网络的BP自己是大概清楚的，但是如果要写出来，就发现不知如何下笔，重新看了别人写的推导又发现自己对矩阵求导的一些符号标记不是特别清楚。这里将矩阵微分法和自己之前不是很清楚的规范的符号标记记录一下。完整的规则可以查看Matrix Differentiation.

1. 常用符号和术语

  • 在公式中，向量表示只有一列的矩阵，即向量通常表示为列向量
  • 包含多列的向量（矩阵）通常使用大写的粗体大写字母来表示${\bf A}, {\bf B}, {\bf X}$。而单列的矩阵（向量）通常使用粗体小写字母来表示。${\bf a}, {\bf b}, {\bf x}$并且约定俗称的是，通常使从字母表靠前顺序的字母来作为已知的矩阵，而使用字母表靠后顺序的字母作为未知的或者是变量矩阵。
  • 矩阵的行列式可以使用${\mid \bf A \mid }$或者${det(\bf A)}$去表达， 矩阵的秩表示为($rank(\bf A)$。

2. 矩阵微分法

在矩阵求导中通常采用分子布局来显示求导结果，即对于向量${\bf y} = \begin {bmatrix} y_1 \\ y_2 \\ \vdots \\ y_n\end{bmatrix} $，关于x的求导在分子布局下的表现形式为$\frac {\partial \bf y}{\partial x} = \begin {bmatrix} \frac{\partial y_1}{\partial x} \\ \frac{\partial y_2}{\partial x} \\ \vdots \\ \frac{\partial y_n}{\partial x} \end{bmatrix}\tag{1}$

而在分母布局下为$\frac {\partial \bf y}{\partial x} = \begin {bmatrix} \frac{\partial y_1}{\partial x} & \frac{\partial y_2}{\partial x} & \cdots & \frac{\partial y_n}{\partial x} \end{bmatrix}\tag{2}$

2.1 关于标量的导数

2.2 关于向量的导数

2.2.1 标量关于向量的导数

标量 $y$ 关于向量 ${\bf x} = \begin {bmatrix} x_1 \\ x_2 \\ \vdots \\ x_n \end{bmatrix}$ 的求导可以表示为$\frac {\partial y} {\partial \bf x} = \begin{bmatrix} \frac {\partial y} {\partial x_1} & \frac {\partial y} {\partial x_2} & \cdots & \frac {\partial y} {\partial x_n} \end{bmatrix}\tag{3}$

此时该向量称为梯度向量。

2.2.2 向量关于向量的导数

向量 ${\bf y} = \begin{bmatrix} y_1 \\ y_2 \\ \vdots \\ y_n \end{bmatrix}$ 关于向量 ${\bf x} = \begin{bmatrix} x_1 \\ x_2 \\ \vdots \\ x_n \end{bmatrix}$ 的导数可以可以看作n个标量对于向量求导，求导结果为 $\frac {\partial \bf y}{\partial \bf x} = \begin {bmatrix} \frac{\partial y_1 }{\partial x_1 } & \frac{\partial y_1 }{\partial x_2 }& \cdots & \frac{\partial y_1 }{\partial x_n } \\ \frac{\partial y_2}{\partial x_1 } & \frac{\partial y_2}{\partial x_2 } & \cdots & \frac{\partial y_2}{\partial x_n } \\ \vdots & \vdots & \ddots & \vdots \\ \frac{\partial y_n }{\partial x_1 } & \frac{\partial y_n }{\partial x_2 } & \cdots & \frac{\partial y_n}{\partial x_n } \end{bmatrix}\tag{4}$

此时得到的矩阵叫Jacobian矩阵

caffe2安装踩坑

  1. 官方很贴心的给除了安装过程中的 常见问题帮助
  2. 用cmake来编译时可以通过PYTHON_EXECUTABLE来指定python解释器
  3. 注意编译时的环境和使用的环境要一致，之前一直报 找不到caffe2_pybind11_state_gpu的原因是因为编译时使用的Python解释器环境是Anaconda的默认的python3.6环境，但是又想在py27下使用。
  4. 警告：No handlers could be found for logger “caffe2.python.net_drawer”
    net_drawer will not run correctly. Please install the correct dependencies.
    解决方案：conda install pydot

心得：

  1. PYTHONPATH的位置指明了额外的python模块的地址。
  2. LD_LIBRARY_PATH用来指明需要动态链接库和静态链接库的地址
  3. PATH指明可执行二进制文件的地址
  4. 没有sudo权限的用户在安装软件时要额外注意对这几个地址的更新。

Faster/Mask R-CNN 复现心得和Nuclei detection中训练细节

发表于 2018-05-22 | 分类于 deep learning ， detection |

这段时间一直在做检测/分割相关方面的工作，之前参加了Kaggle上的Nuclei detection比赛，从别人的代码中学到很多东西。为了能够更进一步了解detection 中的细节，决定在Heng Cherkeng代码的基础上复现一下faster/mask rcnn上的结果，这里记录一下复现过程中发现的两者实现的不同之处，以及复现的过程中才了解到的模型训练中的一些细节。

RPN 网络训练：

  1. 给anchors设定标签时，首先是将和gt_goxes的IoU大于一定阈值（0.7）的设为fg，这种情况下仍然有很多的gt_boxes没有对应的anchors，为了给每一个gt_boxes都至少设定一个对应的anchors, 就从所有anchors中选出一个和其IoU最大的anchors作为fg。
    在Heng 的代码中没有做这个工作。

  2. 我们知道，在给初始的anchors指定标签时，是根据anchors和gt_boxes之间的IoU值来指定的，这里的一个指的注意的地方是fg和bg的阈值之间需要有一个间隔（如fg：0.7，bg：0.3），这个间隔应该是很重要的，使得bg和fg之间的边界更加明显一些，有助于网络的收敛。
    在Heng的代码中，训练Nuclei Detection的时候bg和fg的阈值都是0.5。

  3. 在RPN网络训练时也需要设定一张图片中anchors的batch_size和fg和bg的比例（1：1），这个设定有助于避免bg过多导致的ease sample现象，在实际中也发现，fg的数量和bg的实际数量比大概在1：10左右，尽管仍然很大，但如过不设置batch_size的话和比例的话，fg：bg的比例将达到1：20000（对于尺度在1024x1024的图像）。
    Heng Nuclei Detection代码中，没有设置fb和bg的比例，起解决正负样本数量不均的的方法是通过给正负样本设置不同的权重，使得fg_numfg_weight == bg_num bg_weight。 另一个可以尝试的方法是通过使用focal loss去使得被错分的样例权重增加。

  4. 在通过网络之后，得到了所有anchors回归之后的结果，但是不能将其全部都用做后续RCNN的训练，对于RCNN来说，proposal区域应该尽量位于gt_boxes的附近。所以需要选择一部分“质量”比较好的proposal，而选择的依据就是根据RPN网络产生的logits通过NMS操作去选择，这里有两个比较重要的参数TRAIN_RPN_PRE_NMS_TOP_N = 12000 TRAIN_RPN_POST_NMS_TOP_N = 2000因为若直接将所有proposals都用在NMS中，会严重影响模型的效率，因此在进行NMS之前先选出logits最高的12000个anchors，在NMS操作后剩余的anchors中再选出2000个anchors。
    Heng的代码中并没有采用这种方式，而是直接通过一个预先设置的阈值rpn_train_nms_pre_score_threshold=0.7去选择得分大于这个数值的anchors。

  5. RPN在训练时回归网络时，通过RPN_BBOX_STD_DEV = np.array([0.1, 0.1, 0.2, 0.2])去将四个坐标的尺度去进行放大，目前不知道不使用放大对后续的训练有没有影响。
    Heng 模型中没有进行这个处理。
